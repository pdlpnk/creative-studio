# Naming Conventions

## Идентификаторы

| Сущность | Формат | Пример |
|---|---|---|
| Концепция | `CC-NNNN` | `CC-0001` |
| Вариация концепции | `CC-NNNN-VNN` | `CC-0001-V01` |
| Тест | `TEST-NNNN` | `TEST-0001` |
| Знание | `LRN-NNNN` | `LRN-0001` |
| Промпт | `PRM-NNNN` | `PRM-0001` |

Идентификаторы постоянны, не переиспользуются и не меняются при переименовании сущности. Нумерация дополняется ведущими нулями до указанной длины.

## Файлы и текст

- Названия файлов и каталогов пишутся в `lowercase kebab-case`: например, `product-vision.md`.
- Расширения файлов пишутся строчными буквами.
- Markdown-файлы хранятся в кодировке UTF-8.
- Название документа начинается с одного заголовка первого уровня (`#`).
- Внутренние ссылки по возможности используют относительные пути.
