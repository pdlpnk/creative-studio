# Marketing Library

Marketing Library makes Designer choose the advertising reason before selecting the visual. It is a local source inside the existing Designer Agent, not a new agent or analytics module.

The files use JSON syntax, which is valid YAML 1.2 and can be loaded with Python's standard library. The selection sequence is:

`Creative Idea → Marketing Hook → Emotion → Visual Trigger → Primary Object → Supporting Objects → Composition → Visual Style → Final Prompt`.

`offers.yaml` sets the hard boundary: approved text is copied exactly; amounts, bonuses, cashback, USDT, deadlines, testimonials, statistics and financial claims are never invented. Hooks requiring those facts are unavailable unless the Creative Plan explicitly confirms them.

All audience profiles are adult-only. The library excludes minors, fake social proof, guaranteed outcomes, no-risk claims, celebrity likenesses and protected brands without supplied materials.
