# Creative Reference Library

## Назначение

Каталог `assets/references` предназначен для:

- хранения старых рекламных креативов;
- формирования датасета для последующего анализа;
- построения Creative Concepts на основе накопленных материалов;
- хранения креативов с результатами `winner`, `loser`, `average` и `unknown`.

Проигравшие и не классифицированные материалы являются частью истории экспериментов и сохраняются наравне с победителями.

## Распределение файлов

- `registration/tr` — креативы для регистраций на турецком GEO;
- `registration/az` — креативы для регистраций на азербайджанском GEO;
- `lead/tr` — лидовые креативы для Турции;
- `lead/az` — лидовые креативы для Азербайджана;
- `unclassified` — файлы, которые пока невозможно уверенно классифицировать.

После уточнения GEO и funnel файл из `unclassified` следует переместить в соответствующий каталог и привести его название к установленному формату.

## Названия файлов

Формат:

```text
<geo>-<funnel>-<result>-<number>.<extension>
```

Примеры:

- `tr-registration-winner-001.jpg`
- `az-registration-unknown-002.png`
- `tr-lead-loser-003.jpeg`
- `az-lead-unknown-004.webp`

Допустимые значения:

| Часть имени | Значения |
|---|---|
| GEO | `tr`, `az` |
| Funnel | `registration`, `lead` |
| Result | `winner`, `loser`, `average`, `unknown` |

`number` — уникальный трёхзначный порядковый номер файла в выбранной комбинации GEO и funnel. Названия записываются строчными буквами без пробелов.

## Правила работы с файлами

- Сохранять оригинальное изображение без повторного сжатия.
- Не менять размер старых изображений.
- Новые креативы проекта создавать только в формате `1080×1080`.
- Использовать только расширения `jpg`, `jpeg`, `png` и `webp`.
- Не хранить PSD, ZIP, видео и временные файлы.
- Не удалять проигравшие креативы.
- Не загружать дубликаты.
- Не хранить API-ключи, персональные данные, учётные данные и другие секреты.

Перед добавлением файла необходимо проверить, что такое же изображение ещё не хранится в библиотеке, а его имя, GEO, funnel и результат указаны корректно.

## Git LFS

Растровые изображения должны отслеживаться через Git LFS, когда Git LFS доступен в рабочей среде репозитория. Целевые шаблоны отслеживания:

- `*.png`
- `*.jpg`
- `*.jpeg`
- `*.webp`

Если Git LFS недоступен, его установка и настройка выполняются отдельно и не должны происходить автоматически при подготовке структуры библиотеки.
