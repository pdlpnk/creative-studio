# Creative Concept Cards

Concept Card — структурированное описание одной креативной гипотезы. Карточка фиксирует стратегический замысел, аудиторию, предложение, визуальное направление, текстовые элементы и то, что необходимо проверить.

Concept Card не равна готовому изображению. Она служит постановкой задачи для производства: из одной карточки могут создаваться несколько визуальных или текстовых вариаций, сохраняющих исходную гипотезу.

Карточки являются источником структурированного контекста для будущего Prompt Engine. На текущем этапе этот каталог только хранит знания и не генерирует промпты или изображения.

## Правила

- Каждая Concept Card должна иметь уникальный и постоянный ID.
- Одна карточка описывает одну основную креативную гипотезу.
- Вариации должны ссылаться на родительский концепт через `parent_id`.
- Карточки не удаляются после неудачного теста: отрицательные результаты сохраняют ценность для обучения.
- Статусы `winner` и `loser` используются только после тестирования.
- До получения результата карточка проходит статусы жизненного цикла, начиная с `idea`.
- Фактические утверждения, оффер и тексты должны быть проверяемыми и не вводить аудиторию в заблуждение.

## Именование

| Сущность | Формат | Пример |
|---|---|---|
| Concept | `CC-NNNN` | `CC-0001` |
| Variant | `CC-NNNN-VNN` | `CC-0001-V01` |
| Файл концепта | `CC-NNNN.md` | `CC-0001.md` |

ID и имя файла должны совпадать. Шаблон новой карточки находится в [`template.md`](template.md), а примеры — в каталоге [`examples`](examples/).
